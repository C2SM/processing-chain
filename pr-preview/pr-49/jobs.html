Overview

The jobs described here are available for use in the processing chain. For every target, you can choose from a list of available jobs. As some jobs depend on the result of others, the order indicated here has to be respected.

COSMO:

  1. jobs.biofluxes.main() | jobs.prepare_data.main() | jobs.emissions.main() | jobs.online_vprm.main() | jobs.oae.main() | jobs.obs_nudging.main()

  2. jobs.int2lm.main()

  3. jobs.post_int2lm.main()

  4. jobs.octe.main()

  5. jobs.cosmo.main()

  6. jobs.check_output.main()

  7. jobs.reduce_output.main()

  8. jobs.post_cosmo.main() | jobs.verify_chain.main()

COSMOART:

  1. jobs.emissions.main() | jobs.prepare_data.main() | jobs.photo_rate.main() | jobs.obs_nudging.main() | jobs.online_vprm.main() | jobs.oae.main()

  2. jobs.int2lm.main()

  3. jobs.cosmo.main()

  4. jobs.check_output.main()

  5. jobs.reduce_output.main()

  6. jobs.post_cosmo.main() | jobs.verify_chain.main()

ICON:

  1. jobs.prepare_data.main()

  2. jobs.icon.main()

ICONART:

  1. jobs.prepare_data.main()

  2. jobs.icon.main()

ICONARTOEM:

  1. jobs.prepare_data.main()

  2. jobs.oae.main()

  3. jobs.icon.main()

Adding new jobs

Adding a new job to the chain is simple:

  1. In the directory jobs/, create a file called <jobname>.py containing a function called main which takes the same arguments as every other job. Make sure the function is documented with a docstring.

  2. Import it in jobs/__init__.py to make it accessible to run_chain.py.

  3. Add the function to the documentation. You find the file describing this page at doc/source/jobs.rst.
